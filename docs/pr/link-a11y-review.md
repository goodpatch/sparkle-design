## 対応内容（修正が必要な場合）

### Fail項目: なし

現時点でFailと判定される項目はありません。

### Needs review項目: 4件

#### 1) フォーカスインジケータの明示的スタイル（1.4.11 / 2.4.7 / 2.4.13）

- **現状**: `index.tsx` にフォーカス時のスタイル（`focus-visible`, `outline` 等）が明示的に定義されていない。ブラウザデフォルトのフォーカスリングに依存している。
- **影響**: ブラウザデフォルトのアウトラインは最低限の視認性を提供するが、デザインシステムとして一貫したフォーカス表現とは言えない。
- **推奨対応**: Buttonコンポーネントで実装済みの `focus-visible:ring-2 ring-offset-2` パターンに倣い、Linkにも明示的なフォーカスリングスタイルを追加する。Figma/デザインシステム側のフォーカス仕様を確認の上で対応。
- **該当箇所**: `src/components/ui/link/index.tsx:68`

#### 2) ターゲットサイズ（2.5.8）

- **現状**: インラインリンクとして表示されるため、テキスト量やフォントサイズによっては24x24px未満となりうる。
- **影響**: WCAG 2.5.8の例外規定「インラインテキスト内のリンクはターゲットサイズ要件の例外に該当する」に該当する可能性が高い。
- **推奨対応**: 現状維持で問題ないが、デザインチーム確認を推奨。

## 補足

### 将来的な改善候補

1. **フォーカスリングスタイルの追加**: Buttonコンポーネントと一貫した `focus-visible` スタイルの定義（1.4.11 / 2.4.7 / 2.4.13 対応）
2. **Down系ハンドラの `@deprecated` 化**: プロジェクトポリシー（WCAG 2.5.2）に従い、`onMouseDown` / `onPointerDown` / `onTouchStart` をButtonと同様に `@deprecated` 化する検討
3. **`isOpenInNew` 時の支援技術への通知**: 外部リンクアイコン（`open_in_new`）は `aria-hidden="true"` で正しく隠蔽されているが、「新しいタブで開く」旨をスクリーンリーダーに伝える仕組み（例: `sr-only` テキストの自動付与）の検討。現状は利用者が `aria-label` 等で明示する責務がある
4. **`href` 未指定時の警告**: `href` なしの `<a>` は暗黙のrole="link"を持たないため、開発時の `console.warn` による注意喚起の検討
