## コンポーネント概要

`Icon` は Material Symbols 等のアイコンフォントを `<span>` 要素で表示する装飾用コンポーネントである。デフォルトで `aria-hidden="true"` が付与されており、支援技術からは隠される。インタラクティブな要素ではなく、純粋な視覚的装飾として機能する。

主な特徴:
- `<span>` 要素を使用（非インタラクティブ）
- `aria-hidden="true"` がデフォルトで付与
- `select-none` クラスでテキスト選択を防止
- `icon` prop でアイコン名（テキスト）を表示
- `size` / `fill` prop でタイポグラフィクラスを制御

## 補足

1. **装飾アイコンとしての設計が適切**: `aria-hidden="true"` がデフォルトで付与されているため、スクリーンリーダーは本コンポーネントを無視する。意味のある情報を持つアイコンとして使う場合は、利用者側が親要素（例: `<button aria-label="お気に入りに追加"><Icon icon="favorite" /></button>`）でアクセシブルネームを提供する責務を持つ。この設計はWCAG 1.1.1の装飾画像パターンに合致する。

2. **`...props` spread によるオーバーライド**: `aria-hidden` は `{...props}` より前に記述されているため（`index.tsx:56-57`）、利用者が `aria-hidden="false"` を渡した場合にオーバーライド可能。これは意味のあるアイコンとして使うケースへの拡張性を持たせている。ただし、その場合はアクセシブルネーム（`aria-label` 等）も併せて設定する必要がある点に注意。

3. **N/A項目が多い理由**: `Icon` は非インタラクティブな装飾用 `<span>` 要素であり、キーボード操作、フォーカス管理、フォーム入力、ポインタ操作等に関するWCAG基準の大半が該当しない。これは設計として正しい。

4. **テストカバレッジ**: アクセシビリティに関するテストは `aria-hidden` の検証を含め十分にカバーされている。
