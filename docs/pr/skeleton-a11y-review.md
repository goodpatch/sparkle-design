## 対応内容（修正が必要な場合）

### Needs review: ローディング状態のセマンティクス付与 (1.1.1 / 4.1.2 / 4.1.3)

**現状の問題:**
- Skeletonコンポーネントは `<div>` 要素を使用しているが、`role="status"`, `aria-busy="true"`, `aria-label` がデフォルトで付与されていない
- 支援技術（スクリーンリーダー等）がこの要素をローディングプレースホルダーとして認識できない
- 視覚的には `animate-pulse` アニメーションで読み込み中であることを伝えているが、非視覚的なチャネルでは伝達されない

**推奨修正:**

コンポーネント単体への修正と、利用側での対応の2つのアプローチが考えられる。

#### アプローチA: コンポーネントにデフォルトのARIA属性を付与

1. `<div>` にデフォルトで `role="status"` を付与する
2. `aria-busy="true"` をデフォルトで付与する（ローディング中であることを示す）
3. `aria-label` のデフォルト値として「読み込み中 (Loading)」等を設定する（任意）
4. `{...props}` による上書きは引き続き可能にする

**注意:** 複数のSkeletonを同時に表示するケース（`index.stories.tsx:38-48` ProfileCard, `index.stories.tsx:53-63` ImageCard）では、各Skeletonが個別に `role="status"` を持つと冗長になる可能性がある。その場合は、親要素に `role="status"` + `aria-busy="true"` を付与し、個別のSkeletonは装飾的要素として扱うアプローチが望ましい。

#### アプローチB: 利用側ガイドラインの整備（推奨）

Skeletonは低レベルなビジュアルプリミティブであるため、セマンティクスの責務を利用側に委ねる設計も妥当である。

1. Storybookのドキュメントに、アクセシブルな使用例を追加する
2. 例: 親要素に `role="status"` + `aria-busy="true"` + `aria-label="コンテンツを読み込み中"` を付与するパターンを推奨する
3. 既存テスト (`index.test.tsx:124-152`) のAccessibilityセクションを参考に、ベストプラクティスを文書化する

**影響度:** 中程度 --- ローディング状態の視覚的な伝達はアニメーションで行われているが、スクリーンリーダーユーザーにはローディング中であることが伝わらない。ただし、Skeleton単体よりも利用側のコンテキスト（ページ全体のローディング状態管理等）で対応するケースが一般的である。

## 補足

### コンポーネント特性

Skeletonは非インタラクティブなローディングプレースホルダーコンポーネントであるため、WCAG チェックリスト全32項目のうち大部分（24項目）が N/A 判定となる。これはコンポーネントの性質上妥当である。

| N/A | 24 |

### Skeletonのアクセシビリティに関する一般的なパターン

ローディングプレースホルダー（Skeleton）のアクセシビリティ対応には、業界で以下のパターンが一般的に採用されている:

1. **コンテナレベルでのステータス通知**: 個別のSkeleton要素ではなく、それを含むコンテナに `role="status"` + `aria-busy="true"` を付与
2. **ライブリージョンの活用**: `aria-live="polite"` を使用し、読み込み完了時にコンテンツの変化を支援技術に通知
3. **スクリーンリーダー専用テキスト**: `sr-only` クラスで「読み込み中」等のテキストを追加

本コンポーネントは低レベルなビジュアルプリミティブとして設計されているため、アプローチBのように利用側ガイドラインの整備で対応するのが、コンポーネント設計の一貫性を保ちつつアクセシビリティを確保する現実的な方法と考えられる。

### Storybook での確認

`index.stories.tsx` では3つのストーリー（Default, ProfileCard, ImageCard）が定義されており、基本的な視覚的確認が可能。ただし、いずれのストーリーにもARIA属性（`role`, `aria-label`, `aria-busy`, `aria-live`）が含まれていないため、アクセシブルな使用例のストーリー追加を検討することを推奨する。
