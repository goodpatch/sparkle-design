## 対応内容

以下の修正を実施済みです。

### 修正済み Fail 項目

#### 1) 1.1.1 非テキストコンテンツ / 2.5.3 ラベルを含む名前 / 4.1.2 名前・役割・値 — 閉じるボタンのアクセシブルネーム欠如 → **修正済み**

- **問題**: 閉じるボタン（IconButton）に `aria-label` が設定されておらず、スクリーンリーダーや音声入力で操作できなかった
- **対応**: `aria-label="閉じる"` を IconButton に付与

```tsx
// 適用された修正
<IconButton
  icon="close"
  size="sm"
  variant="ghost"
  theme="neutral"
  aria-label="閉じる"
  onClick={() => {
    sonnerToast.dismiss(id);
  }}
/>
```

#### 2) 4.1.3 状態メッセージ — 通知が支援技術に伝わらない → **修正済み**

- **問題**: トーストの通知内容が支援技術に適切に伝わるか不明だった
- **対応**: トースト本体に `role="status"` を付与。暗黙的に `aria-live="polite"` が適用され、トースト表示時に支援技術に通知される

```tsx
// 適用された修正
<div
  role="status"
  className={cn(
    "shadow-float px-3 py-2 ...",
    className
  )}
>
```

### Needs review 項目

#### 3) 2.1.1 キーボード

- **状況**: トーストは `aria-live` 領域であり、自動的にフォーカスを奪わない設計。閉じるボタンは IconButton でキーボード操作可能だが、トースト自体への Tab キーでのフォーカス移動は未検証
- **推奨**: Storybook でキーボード操作を検証

#### 4) 1.4.13 ホバー又はフォーカスで表示されるコンテンツ

- **状況**: トーストは自動表示であり、ホバー/フォーカストリガーではない。閉じるボタンにホバー時の特別な挙動があるかは未確認
- **推奨**: Storybook で閉じるボタンのホバー状態を確認
