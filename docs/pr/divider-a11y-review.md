## 対応内容（修正が必要な場合）

### Needs review: `role="separator"` のデフォルト付与 (1.1.1 / 4.1.2)

**現状の問題:**
- Dividerコンポーネントは `<div>` 要素を使用しているが、`role="separator"` がデフォルトで付与されていない
- 支援技術（スクリーンリーダー等）がこの要素を区切り線として認識できない
- HTML5 `<hr>` 要素はデフォルトで暗黙の `role="separator"` を持つが、`<div>` にはその暗黙ロールがない

**推奨修正:**

1. `<div>` にデフォルトで `role="separator"` を付与する
2. `direction` prop の値に応じて `aria-orientation` を自動設定する:
   - `direction="horizontal"` の場合: `aria-orientation="horizontal"`（`separator` のデフォルト値なので省略可）
   - `direction="vertical"` の場合: `aria-orientation="vertical"`
3. `{...props}` による上書きは引き続き可能にする（現行のカスタム属性テストの互換性維持）

**影響度:** 中程度 — 装飾的な区切り線のため、ユーザー操作に直接的な影響はないが、セマンティクスの正確性とスクリーンリーダーでの文脈理解の向上に寄与する。

## テスト / 検証

### 既存テストの確認

- `index.test.tsx:349-364` にて `role="separator"`, `aria-orientation="horizontal"`, `aria-label="Content separator"` の手動付与テストが存在
- このテストにより、コンポーネントが ARIA 属性のパススルーに対応していることが確認済み

### 修正後に追加すべきテスト

1. デフォルト状態で `role="separator"` が付与されていることの検証
2. `direction="horizontal"` 時に `aria-orientation` が適切に設定されることの検証
3. `direction="vertical"` 時に `aria-orientation="vertical"` が設定されることの検証
4. props で `role` を上書きした場合に上書き値が優先されることの検証

## 補足

### コンポーネント特性

Dividerは非インタラクティブな装飾コンポーネントであるため、WCAG チェックリスト全32項目のうち大部分（24項目）が N/A 判定となる。これはコンポーネントの性質上妥当である。

### 主要な検出事項のまとめ

| 種別 | 件数 |
|---|---:|
| Pass | 6 |
| Fail | 0 |
| Needs review | 2 (1.1.1, 4.1.2 — いずれも同一の `role="separator"` 未付与に起因) |
| N/A | 24 |

### HTML `<hr>` vs `<div>` の選択について

本コンポーネントが `<div>` を使用している理由として、`<hr>` 要素ではスタイリングの制約（特に vertical 方向や dashed スタイルの実装）があることが推察される。`<div>` を使用する場合は、`role="separator"` を明示的に付与することでセマンティクスを補完する必要がある。

### Storybook での確認

`index.stories.tsx` では5つのストーリー（Default, Vertical, Emphasis, LineStyle, Direction）が定義されており、全バリアントの視覚的確認が可能。ただし、Storybook 上でのアクセシビリティテスト（`@storybook/addon-a11y` 等）の適用状況は本レビューのスコープ外。
